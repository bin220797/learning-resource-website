# 项目全面检查与修改方案

## 一、项目现状分析

### 1. 端口配置问题
- **当前状态**：服务器运行在 80 端口（`server.js:316`）
- **问题**：可能与 PHPStudy 的默认服务端口冲突

### 2. 资源更新问题
- **当前状态**：
  - 后台添加资源后，localhost 可以访问
  - 外网访问（notes.bin220797.top）未更新新增加内容
  - 下载显示文件不存在
- **原因分析**：
  - 可能存在缓存问题
  - 可能存在文件路径配置问题
  - Cloudflare 穿透配置可能需要调整

### 3. 系统架构
- **前端**：HTML + CSS + JavaScript
- **后端**：Node.js + Express
- **存储**：支持文件存储和数据库存储（MySQL）
- **文件上传**：支持多种文件类型，最大 5GB

## 二、修改方案

### 1. 端口配置优化

#### 方案 A：修改 Node.js 服务器端口
- **操作**：将 `server.js` 中的端口改为 3000 或其他端口
- **优势**：配置简单，不影响 PHPStudy 服务
- **劣势**：访问 URL 需要添加端口号

#### 方案 B：修改 PHPStudy Nginx 端口并设置反向代理
- **操作**：
  1. 将 PHPStudy Nginx 端口改为 8080
  2. 配置 Nginx 反向代理，将 80 端口请求转发到 Node.js 服务器
- **优势**：访问 URL 不需要端口号，保持原有访问方式
- **劣势**：配置相对复杂

**推荐方案**：方案 B，因为可以保持原有访问方式的一致性。

### 2. 资源更新与下载问题修复

#### 2.1 缓存问题解决
- **前端缓存**：
  - 检查并优化前端缓存策略
  - 确保资源加载时使用正确的缓存控制头
- **Cloudflare 缓存**：
  - 配置 Cloudflare 缓存规则，避免缓存 API 响应
  - 为静态资源设置合理的缓存时间

#### 2.2 文件路径问题修复
- **绝对路径与相对路径**：
  - 确保所有资源路径使用相对路径
  - 检查 `downloadResource` 函数中的路径处理
- **文件存在性检查**：
  - 增强文件存在性检查逻辑
  - 提供更清晰的错误提示

#### 2.3 数据同步问题
- **确保数据一致性**：
  - 检查 `sync-resources.js` 脚本功能
  - 确保后台添加资源后数据正确同步
